# AI 多模态智能模拟面试评测平台：需求分析（第一阶段）

## 1. 建设目标与边界

建设可部署的企业级模拟面试平台，支持管理端配置招聘资产、候选人完成多模态 AI 面试、面试官复核评价和平台生成可追溯报告。

本项目不将 OpenAI、Whisper、TTS 的密钥暴露至浏览器；所有 AI 调用、音视频处理与报告生成均由后端执行并记录审计信息。视频通话本身与视频文件分析分开建设：本期交付浏览器采集与上传、异步分析；实时多人音视频通话需独立 WebRTC 信令与 TURN 服务，作为部署扩展能力。

## 2. 角色与权限

| 角色 | 核心权限 | 默认入口 |
| --- | --- | --- |
| 管理员（ADMIN） | 用户、角色、岗位、题库、面试、报告、统计、系统配置全量管理 | 管理工作台 |
| 面试官（INTERVIEWER） | 查看被分配面试、启动/结束面试、查看答案、人工评分与确认报告 | 面试工作台 |
| 候选人（CANDIDATE） | 查看本人面试、进入面试间、文本/音频/视频作答、查看已发布报告和历史记录 | 我的面试 |

权限采用 RBAC：用户与角色多对多；接口由 Spring Security 方法级授权和业务归属校验双重控制。

## 3. 核心业务流程

1. 管理员维护岗位、题库分类与题库；可创建或审核 AI 生成题目。
2. 管理员创建面试，指定候选人、面试官、岗位、时间、时长和题目策略。
3. 面试官在时间窗口内开始面试；候选人进入独立面试间。
4. 候选人提交文字、录音或视频答案；服务端保存原文件、转写结果和结构化答案。
5. AI 根据岗位能力模型、题目、回答和转写文本完成评分、追问与总结；失败任务可重试且保留错误记录。
6. 面试结束后，面试官可人工复核；系统生成报告，管理员/面试官发布，候选人仅查看已发布报告。
7. 管理员查看按岗位、题库、时间、候选人维度统计的数据看板。

## 4. 功能清单与验收标准

| 模块 | 必须能力 | 完成判定 |
| --- | --- | --- |
| 认证与 RBAC | 登录、注册、刷新令牌、退出、密码加密、接口授权 | 三种角色不可越权访问接口与页面 |
| 用户/角色/岗位 | 分页查询、增改停用、角色分配、岗位能力模型 | 所有写操作落库并记录操作人 |
| 题库 | 分类、题库、题目、难度、标签、导入/AI 生成 | 可按岗位和标签选择有效题目 |
| 面试 | 创建、改期、取消、开始、结束、历史查询 | 状态机与参与人校验正确 |
| 面试间 | 题目编排、倒计时、自动保存、文字/音频/视频作答 | 重新进入可恢复已保存答案 |
| AI 服务 | 出题、追问、转写、TTS、评分、评价、总结 | 请求可追踪、失败可重试、结果可人工覆盖 |
| 报告与统计 | 报告生成/发布/PDF、历史记录、统计分析 | 候选人仅见已发布的本人报告 |
| 运维 | Swagger/Knife4j、健康检查、日志、Docker Compose、Nginx | 新环境可按文档启动并通过健康检查 |

## 5. 非功能性要求

- 安全：BCrypt 密码、JWT 短期访问令牌、敏感配置环境变量化、上传类型/大小校验、审计日志。
- 数据：MySQL 8 外键与索引；软删除字段使用 `NULL` 表示未删除；所有时间使用统一时区策略。
- 性能：列表接口分页；AI、转写、视频分析和 PDF 生成异步化；Redis 用于缓存、限流和任务状态。
- 可观测性：统一错误码、请求 ID、结构化日志、Actuator 健康检查。
- 测试：关键 Service 单元测试、Controller 集成测试、权限回归测试和前端构建测试。

## 6. 数据与初始化要求

保留既有核心表，并在数据库设计阶段补充：岗位、题库分类、媒体文件、AI 会话/消息、AI 任务、操作审计、刷新令牌与统计所需表。

初始化数据目标：管理员 5 人、面试官 20 人、候选人 100 人；Java 300 题、前端 300 题、算法 200 题、数据库 200 题、AI 100 题；100 场合理的模拟面试及关联答案、评价和报告。所有初始密码使用 Spring Security BCrypt。

## 7. 当前工程差距

当前工程已具备基础 JWT、题库、面试、作答、评价和报告雏形；需要补齐 Pinia 状态管理、用户/角色/岗位后台、题库分类、分页规范、OpenAI 适配层、媒体上传与对象存储、AI 异步任务、Swagger/Knife4j、完整 Docker/Nginx、测试和大规模真实测试数据。

## 8. 第二阶段设计输入

系统设计阶段将确定模块边界、包结构、权限模型、状态机、AI 任务模型、对象存储和 Docker 拓扑。默认采用 MinIO（S3 兼容）存储录音/视频/PDF，Redis 作为缓存、限流与异步任务状态存储；OpenAI 密钥仅由后端环境变量读取。
